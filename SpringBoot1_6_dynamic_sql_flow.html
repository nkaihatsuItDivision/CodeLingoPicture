動的SQLの処理フロー

条件に応じた柔軟なクエリ構築の自動化メカニズム

🎯 動的SQL処理の概要

動的SQLとは、実行時の条件に応じてSQL文を自動的に組み立てる仕組みです。

name
学生名検索
departmentId
学部指定
minGpa
最低GPA
maxGpa
最高GPA
入力パラメータ
name = "田中"
departmentId = 1
minGpa = null
maxGpa = 3.5
MyBatis
処理
自動生成SQL
WHERE name LIKE '%田中%'
AND department_id = 1
AND gpa <= 3.5
🔧 動的SQLタグの仕組み
<select id="searchStudents" resultType="Student">
SELECT * FROM students
<where>
<if test="name != null and name != ''">
AND name LIKE CONCAT('%', #{name}, '%')
</if>
<if test="departmentId != null">
AND department_id = #{departmentId}
</if>
<if test="minGpa != null">
AND gpa >= #{minGpa}
</if>
<if test="maxGpa != null">
AND gpa <= #{maxGpa}
</if>
</where>
ORDER BY student_number
</select>
🏷️ <if> タグ
条件式がtrueの場合のみSQL文に含める
🏷️ <where> タグ
WHERE句を自動調整：不要なANDを削除、条件なしの場合WHERE句自体を省略
🏷️ <foreach> タグ
配列やListの要素を繰り返し処理（IN句等で使用）
📋 実際の処理例
シナリオ1: 全条件指定
入力
SearchQuery query = new SearchQuery();
query.setName("田中");
query.setDepartmentId(1L);
query.setMinGpa(3.0);
query.setMaxGpa(4.0);
生成SQL
SELECT * FROM students
WHERE name LIKE '%田中%'
AND department_id = 1
AND gpa >= 3.0
AND gpa <= 4.0
ORDER BY student_number;
シナリオ2: 部分条件指定
入力
SearchQuery query = new SearchQuery();
query.setName("佐藤");
// その他はnull
生成SQL
SELECT * FROM students
WHERE name LIKE '%佐藤%'
ORDER BY student_number;
シナリオ3: 条件なし
入力
SearchQuery query = new SearchQuery();
// 全てnullまたは空文字
生成SQL
SELECT * FROM students
ORDER BY student_number;
WHERE句が自動的に省略される
シナリオ4: 配列条件（foreach）
入力
List<Long> deptIds = Arrays.asList(1L, 2L, 3L);
XML定義
<foreach item="id" collection="departmentIds"
open="department_id IN ("
separator="," close=")">
#{id}
</foreach>
生成SQL
department_id IN (1, 2, 3)
🚀 動的SQLの威力
1つのXML定義で無数の検索パターンに対応！
複雑な条件分岐をXMLタグが自動処理

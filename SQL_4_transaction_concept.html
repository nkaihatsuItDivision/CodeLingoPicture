トランザクション概念図：ACID特性と処理フロー

トランザクションとは？
複数のSQL操作をひとまとまりとして扱い、
「全て成功」または「全て失敗」を保証する仕組み
A
Atomicity
原子性
操作は分割不可能。全て成功するか、全て失敗するかのどちらか。
C
Consistency
一貫性
データの整合性が保たれ、矛盾のない状態を維持する。
I
Isolation
分離性
他のトランザクションと独立して実行され、互いに影響しない。
D
Durability
永続性
確定した変更は永続化され、システム障害でも失われない。

💡 実践例：学生情報の一括更新
-- トランザクション開始
BEGIN;

-- 学生の年齢を更新
UPDATE student
SET age = 21
WHERE student_id = 'S001';

-- 学部も同時に更新
UPDATE student
SET department = '情報工学部'
WHERE student_id = 'S001';

-- 全て成功したら確定
COMMIT;

-- エラーが発生した場合は取り消し
-- ROLLBACK;

メリット：年齢と学部の更新が両方成功するか、両方失敗するかが保証される

🏗️ Entity層（データモデル層）
データベーステーブルと1対1で対応するJavaクラス。Week 7.9で設計したテーブル構造を完全再現
@Entity
@Table(name = "students")
@Data // Lombok: getter/setter自動生成
@NoArgsConstructor
@AllArgsConstructor
public class Student {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "student_id")
    private Integer studentId;

    @Column(name = "name")
    @NotBlank(message = "名前は必須です")
    private String name;

    @Column(name = "email")
    @Email(message = "正しいメールアドレスを入力してください")
    private String email;

    @Column(name = "enrollment_date")
    private LocalDate enrollmentDate;

    @Column(name = "gpa")
    private Double gpa;

    @Column(name = "department_id")
    private Integer departmentId;
}
2
🗃️ Mapper層（データアクセス層）
データベースとの直接的なやり取りを担当し、SQLを実行してEntityオブジェクトに変換
// StudentMapper.java（インターフェース）
@Mapper
public interface StudentMapper {
    List<Student> findAll();
    Student findById(Integer id);
    void save(Student student);
    void update(Student student);
}
<!-- StudentMapper.xml（SQL実装） -->
<select id="findAll" resultType="Student">
    SELECT student_id, name, email, enrollment_date,
           gpa, department_id
    FROM students
    ORDER BY student_id
</select>

<select id="findById" resultType="Student">
    SELECT * FROM students
    WHERE student_id = #{id}
</select>

🎮 Controller層（プレゼンテーション層）
ユーザーからのリクエストを受け取り、適切なサービスを呼び出してレスポンスを返す
@Controller
@RequestMapping("/")
public class StudentController {

    private final StudentService studentService;

    // 学生一覧表示
    @GetMapping("")
    public ModelAndView index(ModelAndView mv) {
        List<Student> studentList = studentService.findAll();
        mv.setViewName("index");
        mv.addObject("studentList", studentList);
        return mv;
    }

    // 学生詳細表示
    @GetMapping("/{id}")
    public ModelAndView detail(@PathVariable Integer id) {
        Student student = studentService.findById(id);
        // ビューに渡す処理...
    }
}

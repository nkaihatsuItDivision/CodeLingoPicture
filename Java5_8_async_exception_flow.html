⚡ CompletableFuture 非同期例外処理フロー

🚀
1. supplyAsync() - 非同期処理開始
バリデーション処理を非同期で実行。別スレッドで処理開始。
ASYNC
❌
2. Exception発生 - バリデーションエラー
StudentValidationException が発生。RuntimeException でラップして例外伝播。
EXCEPTION
🔄
3. thenCompose() - データベース保存処理
正常時はDB保存を実行。例外時はスキップされる。
ASYNC
📧
4. thenCompose() - メール送信処理
正常時はメール送信を実行。エラー時は警告ログで続行。
RECOVERY
⏰
5. orTimeout() - タイムアウト制御
3秒でタイムアウト。TimeoutException が発生する可能性。
TIMEOUT
🛡️
6. exceptionally() - 例外処理
すべての例外をキャッチ。ログ出力とデフォルト値を返却。
HANDLE
✅
7. Result返却 - 処理完了
正常結果または例外処理結果を呼び出し元に返却。
COMPLETE
基本的な非同期例外処理
public CompletableFuture<String> processAsync(StudentData data) { return CompletableFuture .supplyAsync(() -> { // 1. バリデーション（例外発生可能性） try { validateStudent(data); return data; } catch (StudentValidationException e) { throw new RuntimeException("バリデーションエラー", e); } }) .thenCompose(validData -> { // 2. データベース保存 return CompletableFuture.supplyAsync(() -> { try { return repository.save(validData); } catch (DataAccessException e) { throw new RuntimeException("保存エラー", e); } }); }) .exceptionally(throwable -> { // 3. 例外処理 logger.severe("処理エラー: " + throwable.getMessage()); return "処理失敗"; }); }
タイムアウト付き例外処理
public CompletableFuture<String> processWithTimeout(StudentData data) { return CompletableFuture .supplyAsync(() -> { try { // 時間のかかる処理をシミュレート Thread.sleep(2000); return processStudent(data); } catch (InterruptedException e) { Thread.currentThread().interrupt(); throw new RuntimeException("処理中断", e); } }) .orTimeout(3, TimeUnit.SECONDS) // 3秒でタイムアウト .exceptionally(throwable -> { if (throwable instanceof TimeoutException) { logger.warning("処理タイムアウト: " + data.getId()); return "タイムアウトエラー"; } else { logger.severe("処理エラー: " + throwable.getMessage()); return "処理失敗"; } }); }
部分的失敗を許容する処理
public CompletableFuture<RegistrationResult> registerWithOptionalServices(StudentData data) { // メイン登録処理 CompletableFuture<Student> mainRegistration = registerStudentCore(data); // メール送信（失敗OK） CompletableFuture<Boolean> emailResult = mainRegistration .thenCompose(student -> sendWelcomeEmail(student.getEmail())) .exceptionally(throwable -> { logger.warning("メール送信失敗（続行）: " + throwable.getMessage()); return false; // 失敗でも続行 }); // SMS送信（失敗OK） CompletableFuture<Boolean> smsResult = mainRegistration .thenCompose(student -> sendWelcomeSMS(student.getPhone())) .exceptionally(throwable -> { logger.warning("SMS送信失敗（続行）: " + throwable.getMessage()); return false; // 失敗でも続行 }); // すべての結果を統合 return mainRegistration .thenCombine(emailResult, (student, emailSuccess) -> new RegistrationResult(student, emailSuccess)) .thenCombine(smsResult, (result, smsSuccess) -> { result.setSmsSuccess(smsSuccess); return result; }); }

🎯 非同期例外処理のパターン

🚨 クリティカルエラー（停止）

必須処理での例外は exceptionally() でエラー結果を返し、呼び出し元で判断

⚠️ オプションエラー（続行）

メール送信等の補助処理は警告ログのみで、メイン処理は続行

⏰ タイムアウト制御

orTimeout() で処理時間を制限し、TimeoutException を適切に処理

🔄 リトライ機能

一時的な障害に対してリトライ処理を実装し、堅牢性を向上

📝 詳細ログ出力

非同期処理では原因特定が困難なため、詳細なログ出力が必須

🎭 例外の種類判定

instanceof で例外の種類を判定し、それぞれに適した処理を実行

null概念の理解
正常な状態
String name = "田中";
📦 "田中"
変数が実際の文字列オブジェクトを参照している状態。 メソッドを安全に呼び出せる。
空文字列
String name = "";
📦 ""
長さ0の文字列オブジェクトを参照している状態。 オブジェクトは存在するのでメソッド呼び出し可能。
null状態
String name = null;
∅
何も参照していない状態。 メソッドを呼び出すとNullPointerExceptionが発生。
🏠 身近な例で理解しよう
📋
正常状態
住所が書かれた紙
「東京都新宿区1-1-1」
📄
空文字列
何も書かれていない紙
（紙自体は存在）
❌
null状態
紙自体が存在しない
（手に何も持っていない）
nullで起こる問題（NullPointerException）
❌ 危険なパターン1
String name = null; int length = name.length(); // エラー！
NullPointerException発生
❌ 危険なパターン2
String message = null; boolean result = message.equals("Hello"); // エラー！
NullPointerException発生
nullの安全な扱い方
✅ 対策1: 事前にnullチェック
String name = getUserInput(); if (name != null) { int length = name.length(); // 安全 System.out.println("文字数: " + length); } else { System.out.println("名前が設定されていません"); }
✅ 対策2: nullセーフな比較メソッド
// 危険: message.equals("Hello") // 安全: "Hello".equals(message) String message = null; boolean result = "Hello".equals(message); // false（エラーなし）
✅ 対策3: デフォルト値の設定
String name = getUserInput(); if (name == null) { name = "名前未設定"; // デフォルト値を設定 } System.out.println("こんにちは、" + name + "さん");

短絡評価（ショートサーキット）の動作説明
&&演算子の短絡評価
左の条件がfalseの場合
false && (複雑な処理)
左の条件を評価: false
⬇️
右の条件は評価しない（スキップ）
⬇️
結果: false
左の条件がtrueの場合
true && (右の条件)
左の条件を評価: true
⬇️
右の条件も評価する
⬇️
結果: 右の条件の値
||演算子の短絡評価
左の条件がtrueの場合
true || (複雑な処理)
左の条件を評価: true
⬇️
右の条件は評価しない（スキップ）
⬇️
結果: true
左の条件がfalseの場合
false || (右の条件)
左の条件を評価: false
⬇️
右の条件も評価する
⬇️
結果: 右の条件の値
⚠️ 短絡評価による安全なプログラミング例
int a = 5; int b = 0; // 危険：ゼロ除算エラーが発生する可能性 if (a / b > 2) { // b が 0 だとエラー！ System.out.println("条件成立"); } // 安全：短絡評価でゼロ除算を回避 if (b != 0 && a / b > 2) { // b != 0 がfalseなら a / b は実行されない System.out.println("条件成立"); }
✅ 短絡評価の3つのメリット
1. パフォーマンス向上
不要な計算をスキップして処理を高速化
2. エラー回避
ゼロ除算やnullアクセスなどの実行時エラーを防止
3. 安全なプログラミング
条件チェックの順序で安全性を担保
📝 短絡評価のルール まとめ
&&
左がfalseなら右は評価せずfalse
||
左がtrueなら右は評価せずtrue

⚠️ publicフィールドの危険性

💻 DangerousStudentクラス 

public class DangerousStudent {
    // ❌ 危険：全て public
    public String name;
    public int age;
    public double gpa;
    
    public DangerousStudent(String name, 
                         int age, double gpa) {
        this.name = name;
        this.age = age;
        this.gpa = gpa;
    }
}

⚠️ 外部からの不正アクセス 

DangerousStudent student = 
    new DangerousStudent("田中", 20, 3.5);

// ❌ 直接フィールドを変更可能
student.name = "";        // 空文字
student.age = -5;       // 負の年齢
student.gpa = 99.9;     // 不正なGPA
⬇️ 結果 ⬇️

💥 実行時の異常な状態 

// 出力結果：異常なデータ
名前：、年齢：-5歳、GPA：99.9

// データの整合性が完全に破綻

🚨 publicフィールドが引き起こす問題

❌
データの整合性破綻
年齢-5歳、GPA99.9など、現実的にありえない値が設定される
🐛
予期しないエラーの原因
null参照や範囲外の値により実行時エラーが発生
🔍
デバッグの困難
どこでデータが変更されたか追跡が不可能
🔧
保守の困難
仕様変更時に全ての直接アクセス箇所を修正する必要

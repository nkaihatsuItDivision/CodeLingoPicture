📝 お名前フィールド
required maxlength
HTMLコード
<input type="text" name="name" placeholder="例：山田太郎" maxlength="50" required>
使用機能：
• required: 必須入力チェック
• maxlength="50": 50文字以内制限
• placeholder: 入力例を表示
動作テスト
✅ 空欄で送信 → エラー表示
✅ 51文字入力 → 入力不可
✅ 適切な名前 → 正常動作
🔤 フリガナフィールド
required pattern maxlength
HTMLコード
<input type="text" name="kana" placeholder="例：ヤマダタロウ" pattern="[ァ-ヴー]+" title="カタカナで入力してください" maxlength="50" required>
高度な機能組み合わせ：
• pattern="[ァ-ヴー]+": カタカナのみ許可
• title属性: エラー時のメッセージ
• maxlength + required: 基本制限
動作テスト
❌ "やまだ" → パターンエラー
❌ "yamada" → パターンエラー
✅ "ヤマダタロウ" → 正常
📧 メールアドレスフィールド
required maxlength
HTMLコード
<input type="email" name="email" placeholder="例：yamada@example.com" maxlength="100" required>
type="email"の自動機能：
• メールアドレス形式の自動チェック
• スマホで@キーが出やすい
• 送信前の形式バリデーション
動作テスト
❌ "abc" → 形式エラー
❌ "user@" → 不完全
✅ "user@example.com" → 正常
📞 電話番号フィールド（任意）
pattern
HTMLコード
<input type="tel" name="phone" placeholder="例：090-1234-5678" pattern="[0-9]{2,4}-[0-9]{2,4}-[0-9]{3,4}" title="ハイフン区切りで入力">
任意項目の工夫：
• requiredなし = 空欄でもOK
• 入力時はpatternでチェック
• ハイフン形式を強制
動作テスト
✅ 空欄 → OK（任意項目）
❌ "09012345678" → パターンエラー
✅ "090-1234-5678" → 正常
🔢 年齢フィールド
required min/max
HTMLコード
<input type="number" name="age" placeholder="例：25" min="1" max="120" required>
数値範囲制限：
• type="number": 数値のみ入力
• min="1": 1歳以上
• max="120": 120歳以下
動作テスト
❌ 0 → 範囲外エラー
❌ 150 → 範囲外エラー
✅ 25 → 正常
📋 選択フィールド
required
HTMLコード
<select name="category" required> <option value="">選択してください</option> <option value="product">商品について</option> <option value="service">サービスについて</option> </select>
select要素でのrequired：
• 空のvalue=""のoptionを最初に配置
• 選択しないと送信エラー
• ユーザーに選択を促す
動作テスト
❌ 未選択で送信 → エラー
✅ 任意の項目選択 → 正常
📝 メッセージフィールド
required maxlength
HTMLコード
<textarea name="message" placeholder="お問い合わせ内容を詳しく" maxlength="500" rows="6" required></textarea>
textarea要素でのバリデーション：
• 長文入力が可能
• maxlengthで文字数制限
• placeholderで入力ガイド
動作テスト
❌ 空欄で送信 → エラー
✅ 501文字入力 → 入力不可
✅ 適切な内容 → 正常
🎯 使用したバリデーション機能まとめ
required属性
必須入力項目の指定。空欄での送信を防ぐ
pattern属性
正規表現で入力形式を制限（カタカナ、電話番号など）
maxlength属性
最大文字数制限。長すぎる入力を防ぐ
min/max属性
数値の範囲制限。年齢などの妥当性チェック
type属性
入力タイプ指定（email、tel、numberなど）
placeholder属性
入力ヒントの表示。ユーザーガイダンス
title属性
カスタムエラーメッセージの設定
checkbox required
チェックボックスの必須選択（利用規約同意など）

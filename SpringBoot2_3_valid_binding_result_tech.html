@Valid・BindingResult技術解説

Bean Validation自動実行の仕組みとパラメータ詳細

🔧 技術実装の詳細

@PostMapping("/students")
public String createStudent(
    @Valid @ModelAttribute StudentData studentData,
    BindingResult bindingResult,
    Model model) {

    // 重要：パラメータの順序は厳密に守る
    // 1. @Valid対象オブジェクト
    // 2. BindingResult（必ず直後に配置）
    // 3. その他のパラメータ

    if (bindingResult.hasErrors()) {
        // エラー時の処理
        return "students/form";
    }

    // 正常時の処理
    studentService.save(studentData);
    return "redirect:/students";
}

✨ @Validアノテーション

役割：Bean Validationの自動実行を指示

配置場所：@ModelAttributeの直前

動作：
  • StudentDataクラスのアノテーション（@NotBlank、@Emailなど）を自動実行
  • バリデーション結果をBindingResultに格納
  • 手動バリデーション処理が不要になる
@Valid @ModelAttribute StudentData data

📋 BindingResultパラメータ

役割：バリデーション結果の受け取り

配置場所：@Valid対象パラメータの直後

主要メソッド：
  • hasErrors() - エラー有無判定
  • getFieldErrors() - フィールド別エラー取得
  • getAllErrors() - 全エラー取得
BindingResult bindingResult
if (bindingResult.hasErrors()) { ... }

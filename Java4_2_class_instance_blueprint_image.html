💻 プログラムでの対応関係
🎯 設計図 = クラス（1つ）
// 学生の設計図を定義
public class Student {
    String name;   // 名前のフィールド
    int score;     // 得点のフィールド
    // 機能も定義
    String getGrade() { ... }
    void introduce() { ... }
}
🏠 実体 = インスタンス（複数作成可能）
// 設計図から実際の学生オブジェクトを作成
Student student1 = new Student();
student1.name = "田中太郎";
student1.score = 85;
Student student2 = new Student();
student2.name = "佐藤花子";
student2.score = 92;
Student student3 = new Student();
student3.name = "山田一郎";
student3.score = 76;

🔑 重要な理解ポイント

📐
1つのクラス → 複数のインスタンス
設計図は1つだけ作れば、そこから何個でも実体を作ることができます。
🔒
各インスタンスは独立したデータ
田中さんの得点を変更しても、佐藤さんの得点は変わりません。完全に独立しています。
⚙️
同じ機能を使える
全ての学生インスタンスが getGrade() や introduce() メソッドを使用できます。
🔧
newキーワードで作成
「new Student()」で新しいインスタンスが作成され、独立したメモリ領域が確保されます。

getter/setterの基本概念とアクセス制御
❌ 危険：直接アクセス
Studentクラス
public String name;
public int age;
public double gpa;
↕ 制御なし
// 外部から自由に変更可能
student.name = "";
student.age = -100;
student.gpa = 999.9;
✅ 安全：getter/setter
Studentクラス
private String name;
private int age;
private double gpa;
public String getName()
public void setName(String)
public int getAge()
public void setAge(int)
↕ 制御あり
// メソッド経由で制御
student.setName("田中太郎");
student.setAge(20);
String name = student.getName();
🛡️ setterでの検証例
public void setAge(int age) {
    if (age < 0 || age > 150) {
        throw new IllegalArgumentException("年齢は0-150の範囲で設定してください");
    }
    this.age = age;
}
🔒 制御されたアクセス
外部からの直接変更を防ぎ、メソッド経由での安全なアクセスを実現
✅ データ検証
setterで不正な値をチェックし、オブジェクトの整合性を保証
🔧 柔軟な実装
内部実装の変更や処理の追加が外部に影響しない
📊 情報の計算
getterで計算結果や加工された情報を返すことが可能

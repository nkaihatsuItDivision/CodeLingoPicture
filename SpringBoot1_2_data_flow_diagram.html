🔄 データフロー図（リクエストからレスポンスまで）
📋 学生管理システム「学生一覧表示」の完全データフロー
ユーザーがブラウザで「http://localhost:8080/students」にアクセスしてから、
美しい学生一覧画面が表示されるまでの9ステップを詳しく見てみましょう。
1
ユーザーリクエスト
🌐 ブラウザ
ユーザーがブラウザのアドレスバーに「http://localhost:8080/students」を入力してEnterキーを押下
GET http://localhost:8080/students
Host: localhost:8080
User-Agent: Mozilla/5.0...
2
Controller受信
🎮 StudentController
Spring Bootが@GetMappingアノテーションを認識し、showStudentsメソッドを自動実行
@GetMapping("/students")
public String showStudents(Model model) {
// このメソッドが呼ばれる
}
3
Service層呼び出し
🎮 StudentController
ControllerがService層のfindAll()メソッドを呼び出してビジネスロジックを実行
@Autowired
private StudentService service;

List<Student> students = service.findAll();
4
Mapper層呼び出し
⚙️ StudentService
Service層がMapper層のfindAll()メソッドを呼び出してデータアクセス処理を実行
@Service
public class StudentService {
return mapper.findAll();
}
5
SQL自動実行
🗄️ MyBatis
MyBatisがStudentMapper.xmlに定義されたSQLを自動実行してデータベースにアクセス
<!-- StudentMapper.xml -->
SELECT student_id, name, email, gpa
FROM students
ORDER BY student_id;
6
データベースアクセス
💾 MySQL Database
Week 7で設計したstudentsテーブルから実際のデータを取得してResultSetとして返却
-- Week 7で作成したテーブル
mysql> SELECT * FROM students;
+----+-----------+-------------------+------+
| id | name | email | gpa |
+----+-----------+-------------------+------+
| 1 | 田中太郎 | tanaka@example... | 3.5 |
| 2 | 佐藤花子 | sato@example... | 3.8 |
+----+-----------+-------------------+------+
7
オブジェクト変換
📊 MyBatis + Entity
MyBatisがResultSetを自動的にStudentオブジェクトのListに変換してService → Controllerに返却
// Week 5 Lombokで自動生成
@Data
public class Student {
private Integer studentId;
private String name;
// List<Student>として返却
}
8
Model設定・View指定
🎮 StudentController
Controllerが取得したデータをModelに設定し、表示するテンプレートファイル名を返却
// Viewにデータを渡す
model.addAttribute("students", students);

// テンプレートファイルを指定
return "students/index";
9
HTML生成・レスポンス
🎨 Thymeleaf
Thymeleafがテンプレート + データを使って動的HTMLを生成し、ブラウザに美しい画面をレスポンス
<!-- 最終的に生成されるHTML -->
<tr>
<td>1</td>
<td>田中太郎</td>
<td>tanaka@example.com</td>
<td>3.5</td>
</tr>
💡 データフローの美しさ
このデータフローの美しさは、各層が独立していながら完璧に連携していることです。 ControllerはViewの詳細を知らず、ViewはModelの実装を知らず、ModelはControllerの処理を知りません。 しかし、Spring Bootのアノテーションにより、これらが自動的に連携して動作します。

この疎結合な設計により、例えばModelの実装を変更してもViewやControllerに影響せず、 Viewのデザインを変更してもModelやControllerに影響しません。 これがMVCアーキテクチャの真の価値なのです。

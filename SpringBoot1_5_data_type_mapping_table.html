🚀 実践的な使用例
👤 学生情報の型設計
@Entity
public class Student {
    @Id
    private Long studentId; // BIGINT

    private String name; // VARCHAR
    private Integer age; // INT
    private String email; // VARCHAR
}
📊 成績情報の型設計
@Entity
public class Score {
    private Double gpa; // DOUBLE
    private Integer score; // INT
    private Boolean passed; // BOOLEAN
    private String grade; // VARCHAR
}
📅 日時情報の型設計
@Entity
public class Enrollment {
    private LocalDate enrollmentDate; // DATE
    private LocalDateTime createdAt; // DATETIME
    private LocalDateTime updatedAt; // DATETIME
}
💡 型選択のベストプラクティス
🎯
主キーはLong
自動採番される主キーには必ずLong型を使用。 将来のデータ増加にも対応可能。
📏
String型の長さ指定
@Column(length=50)で適切な長さを指定。 メモリ効率と検索性能が向上。
💰
金額はBigDecimal
金額など精度が重要な計算には 必ずBigDecimal型を使用。
📅
日時型の使い分け
日付のみ→LocalDate 日時両方→LocalDateTime 用途に応じて適切に選択。

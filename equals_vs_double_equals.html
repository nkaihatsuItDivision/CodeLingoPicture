== と .equals() の動作比較
📝 シナリオ1: 文字列リテラル同士の比較
String name1 = "田中"; String name2 = "田中";
name1
→ "田中"
同じオブジェクト
name2
→ "田中"
== での比較
name1 == name2
true
同じオブジェクトを参照しているため
.equals() での比較
name1.equals(name2)
true
内容が同じため
⚠️ シナリオ2: new String で作成した場合
String input = new String("田中"); String stored = "田中";
input
→ "田中" (0x001)
異なるオブジェクト
stored
→ "田中" (0x002)
❌ == での比較
input == stored
false
異なるオブジェクトを参照しているため
✅ .equals() での比較
input.equals(stored)
true
内容が同じため
🔐 シナリオ3: パスワード認証の例
String correctPassword = "secret123"; String userInput = getUserInput(); // ユーザーからの入力
❌ 危険な方法
userInput == correctPassword
多くの場合 false
正しいパスワードでもログイン失敗！
✅ 正しい方法
userInput.equals(correctPassword)
正しく判定
内容で判定するため安全
重要なルール
== 演算子： オブジェクトの参照（住所）を比較
→ 同じオブジェクトかどうかを判定
.equals() メソッド： オブジェクトの内容を比較
→ 値が同じかどうかを判定
文字列比較のベストプラクティス：
常に .equals() を使用する

CRUD操作の具体的な流れ図

StudentMapperの各メソッドが実行される詳細プロセス

CREATE - 学生登録
1
Javaメソッド呼び出し
Serviceクラスから Mapper メソッドを実行
mapper.save(student);
2
MyBatis XML解析
StudentMapper.xmlの<insert>要素を解析
<insert id="save" parameterType="Student">
3
SQL自動生成
EntityオブジェクトからSQL文を自動構築
INSERT INTO students (name, email, birth_date)
VALUES (?, ?, ?)
4
データベース実行
生成されたSQL文をデータベースで実行
自動コミットまたはトランザクション管理
READ - 学生検索
1
検索条件指定
IDまたは検索条件を指定してメソッド呼び出し
Student student = mapper.findById(1L);
2
動的SQL処理
条件に応じてWHERE句を自動構築
<if test="id != null">
AND id = #{id}
</if>
3
SELECT実行
データベースからレコードを取得
SELECT id, name, email, birth_date
FROM students WHERE id = 1
4
Entity変換
取得結果を Student オブジェクトに自動変換
スネークケース→キャメルケース自動変換
UPDATE - 学生更新
1
更新データ準備
変更後のStudentオブジェクトを渡す
mapper.update(modifiedStudent);
2
部分更新SQL生成
変更があったフィールドのみSETに含める
<if test="email != null">
email = #{email},
</if>
3
WHERE条件指定
主キーでレコードを特定
UPDATE students SET email = ?
WHERE id = ?
4
更新実行＆確認
更新件数を返却して処理結果を確認
トランザクション管理による整合性保証
DELETE - 学生削除
1
削除対象指定
削除対象のIDを指定
mapper.delete(1L);
2
関連チェック
外部キー制約による関連データの確認
-- 成績データが存在する場合はエラー
3
物理削除 or 論理削除
設定に応じた削除方式を選択
DELETE FROM students
-- または --
UPDATE students SET deleted = true
4
削除確認
削除件数を返却して処理結果を確認
エラー時は自動ロールバック

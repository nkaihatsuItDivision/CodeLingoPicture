🐛 Eclipse デバッガーの基本使用法

デバッガーとは？

プログラムの実行を一時停止し、変数の値や実行の流れを詳しく調べることができるツールです。バグの発見や、プログラムの動作理解に非常に有効です。

1
ブレークポイント設定
プログラムを停止させたい行の行番号の左側をクリックします。青い丸（●）が表示されれば設定完了です。
public static void addStudent() { System.out.print("学生の名前を入力してください: "); String name = scanner.nextLine(); ● ← ここをクリック // プログラムはここで停止します }
💡 複数の場所にブレークポイントを設定できます
2
デバッグ実行
通常の実行ボタン（▶）の横にある「虫」のアイコンをクリックしてデバッグモードで実行します。
🐛 デバッグボタンの場所：
Eclipse上部のツールバー → 緑の虫アイコン
または右クリック → 「デバッグ」→「Javaアプリケーション」
✅ ブレークポイントで実行が自動停止します
3
ステップ実行
プログラムが停止したら、以下のキーで一行ずつ実行できます。
F6: 次の行に進む（ステップオーバー）
F5: メソッドの中に入る（ステップイン）
F8: 次のブレークポイントまで実行（Resume）
🎯 F6が最もよく使われます
4
変数の確認
デバッグ中は「Variables」ビューで、現在の変数の値をリアルタイムで確認できます。
確認できる情報：
• 各変数の現在の値
• 配列の中身（要素ごと）
• オブジェクトのフィールド
• 式の計算結果
🔍 バグの原因特定に非常に有効です

📺 Eclipse デバッグ画面の構成

🖥️ エディター画面

  • 現在実行中の行が黄色でハイライト
  • ブレークポイントが●で表示
  • コードの内容を確認しながらデバッグ

📊 Variables ビュー

  • 現在の変数の値を表示
  • 変数をクリックして詳細確認
  • 配列やオブジェクトは展開可能

🎮 Debug ビュー

  • 現在のスレッドとメソッド呼び出し履歴
  • 実行制御ボタン（Resume、Step等）
  • デバッグセッションの管理

📝 Console ビュー

  • プログラムの出力結果を表示
  • ユーザー入力（Scanner）も可能
  • エラーメッセージの確認

⌨️ よく使うキーボードショートカット

次の行に進む F6
メソッドに入る F5
実行継続 F8
デバッグ終了 Ctrl+F2
ブレークポイント切替 Ctrl+Shift+B
変数の値表示 Ctrl+Shift+I

🎯 実践例：addStudent() メソッドのデバッグ

public static void addStudent() { if (count >= MAX_STUDENTS) { System.out.println("これ以上登録できません。"); return; } System.out.print("学生の名前を入力してください: "); String name = scanner.nextLine(); // ← ここにブレークポイント設定 System.out.print("点数を入力してください (0-100): "); int score = getValidScore(); names[count] = name; // ← ここでもチェック scores[count] = score; count++; System.out.println("学生を追加しました: " + name + " (" + score + "点)"); }
  1. String name = scanner.nextLine(); の行にブレークポイントを設定
  2. デバッグモードでプログラムを実行
  3. 名前を入力後、プログラムがブレークポイントで停止
  4. Variables ビューで name 変数の値を確認
  5. F6を押して次の行に進む
  6. names[count] = name; の行で再度確認
  7. Variables ビューで names 配列に正しく格納されているかチェック
  8. F8で実行を継続
💡 このデバッグで分かること：
• 入力された値が正しく変数に格納されているか
• 配列への代入が正常に行われているか
• count変数が適切に更新されているか
• 各ステップでのプログラムの状態

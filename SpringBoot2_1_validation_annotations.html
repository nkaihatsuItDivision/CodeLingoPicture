Bean Validation アノテーション

フォーム入力値検証の強力なツール

StudentDataクラスで使用する主要バリデーション
バリデーションアノテーションとは？
Bean Validationアノテーションを使用することで、 複雑な入力チェック処理をアノテーション1つで実現できます。 コードの簡潔性と保守性を大幅に向上させる重要な技術です。
📝
@NotBlank
必須入力項目の検証を行います。null、空文字、空白のみの文字列を不正値として判定します。
使用例
@NotBlank(message = "名前は必須です")
private String name;
適用場面
  • 学生名（必須項目）
  • 住所（必須項目）
  • その他必須文字列項目
📧
@Email
メールアドレス形式の妥当性を検証します。RFC準拠の厳密な形式チェックを自動実行します。
使用例
@Email(message = "有効なメールアドレスを入力してください")
private String email;
適用場面
  • 学生メールアドレス
  • 連絡先メール
  • ログインID（メール形式）
📏
@Size
文字列の長さや配列のサイズを検証します。min・maxパラメータで範囲を指定できます。
使用例
@Size(min = 2, max = 50, message = "名前は2〜50文字で入力してください")
private String name;
適用場面
  • 学生名（2〜50文字）
  • 住所（10〜100文字）
  • パスワード（8〜20文字）
🔢
@Min / @Max
数値の最小値・最大値を検証します。年齢や点数など、数値項目の範囲チェックに使用します。
使用例
@Min(value = 18, message = "年齢は18歳以上で入力してください")
@Max(value = 100, message = "年齢は100歳以下で入力してください")
private Integer age;
適用場面
  • 学生年齢（18〜100歳）
  • テスト点数（0〜100点）
  • 評価値（1〜5段階）
✅
@AssertTrue
boolean値がtrueであることを検証します。利用規約同意など、チェックボックスの必須選択に使用します。
使用例
@AssertTrue(message = "利用規約に同意してください")
private Boolean agreed;
適用場面
  • 利用規約同意
  • 個人情報利用同意
  • 重要事項確認
📅
@NotNull
オブジェクトがnullでないことを検証します。日付や数値など、オブジェクト型フィールドに使用します。
使用例
@NotNull(message = "生年月日は必須です")
private LocalDate birthday;
適用場面
  • 生年月日（必須）
  • 入学日（必須）
  • 選択必須項目
バリデーションによる品質向上効果
🛡️ データ品質保証
不正なデータがデータベースに登録されることを防止し、システムの信頼性を確保
👥 ユーザビリティ向上
分かりやすいエラーメッセージにより、ユーザーが正しい入力を行いやすくなる
⚡ 開発効率向上
複雑な検証ロジックをアノテーション1つで実現し、開発・保守コストを削減
🔧 保守性向上
検証ルールの変更が容易になり、仕様変更への対応が迅速かつ安全に実行可能

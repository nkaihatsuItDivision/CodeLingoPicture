オブジェクトのフィールド値を自動検証する標準化された技術
❌ 従来の手動バリデーション
問題の多い方法
@PostMapping("/students")
public String createStudent(@ModelAttribute StudentData studentData) {
    // 手動でのバリデーション - 問題だらけ！
    List<String> errors = new ArrayList<>();
    
    if (studentData.getName() == null ||
        studentData.getName().trim().isEmpty()) {
        errors.add("名前は必須です");
    }
    if (studentData.getEmail() == null ||
        !studentData.getEmail().contains("@")) {
        errors.add("正しいメールアドレスを入力");
    }
    // このような手動チェックが延々と続く...
}
❌ コードが冗長: バリデーション処理がControllerを圧迫
❌ 保守性が低い: 条件変更時の修正箇所が多数
❌ 一貫性がない: 開発者によってチェック方法が異なる
❌ テストが困難: バリデーション処理の単体テストが複雑
✅ Bean Validation
推奨される方法
@PostMapping("/students")
public String createStudent(
        @Valid @ModelAttribute StudentData studentData,
        BindingResult bindingResult) {
    
    if (bindingResult.hasErrors()) {
        // エラーがある場合、フォーム画面に戻る
        return "students/form";
    }
    
    // バリデーション成功時の処理
    return "redirect:/students";
}
✅ シンプルなコード: @Validだけでバリデーション実行
✅ 宣言的な設計: アノテーションによる分かりやすい検証ルール
✅ 再利用性: 同じDTOクラスを他の場所でも活用可能
✅ 標準仕様: Java標準のため学習コストが低い

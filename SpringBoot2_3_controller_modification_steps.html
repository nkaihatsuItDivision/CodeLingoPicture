Controller改修手順図

Week 9.2版 → Week 9.3版への改修手順

📝 Week 9.2版 Controller
（改修前）
@Controller
@RequestMapping("/students")
public class StudentController {

    @PostMapping
    public String createStudent(
        @ModelAttribute StudentData studentData,
        Model model) {

        // 手動バリデーション処理
        if (studentData.getName().isEmpty()) {
            model.addAttribute("error",
                "名前は必須です");
            return "students/form";
        }

        studentService.save(studentData);
        return "redirect:/students";
    }
}
✨ Week 9.3版 Controller
（改修後）
@Controller
@RequestMapping("/students")
public class StudentController {

    @PostMapping
    public String createStudent(
        @Valid @ModelAttribute StudentData studentData,
        BindingResult bindingResult,
        Model model) {

        // Bean Validation自動実行
        if (bindingResult.hasErrors()) {
            return "students/form";
        }

        studentService.save(studentData);
        return "redirect:/students";
    }
}

🔧 重要な変更点（Step 1）

1️⃣ @Valid追加
@ModelAttributeの直前に@Validを追加
→ StudentDataクラスのアノテーションが自動実行される
2️⃣ BindingResult追加
@ModelAttributeパラメータの直後に追加
→ バリデーション結果を受け取るオブジェクト
3️⃣ 手動チェック削除
if文による手動バリデーション処理を削除
→ Bean Validationが自動で実行
4️⃣ hasErrors()判定
bindingResult.hasErrors()でエラー判定
→ エラー時はフォーム画面に戻る

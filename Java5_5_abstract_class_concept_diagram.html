抽象クラスの概念図

通常のクラス
  • public void displayInfo() { // 具体的な実装あり System.out.println("情報表示"); }
  • public void study() { // 具体的な実装あり System.out.println("学習中"); }
  • public String getType() { // 具体的な実装あり return "一般学生"; }
抽象クラス (abstract)
  • public void displayInfo() { // 具体的な実装あり（共通処理） System.out.println("情報表示"); }
  • public abstract void study();
  • public abstract String getType();
✅ 通常のクラス - インスタンス化可能
// 直接インスタンス化できる Student student = new Student(); student.study(); // "学習中" が出力
コンパイル成功 - 実行可能
❌ 抽象クラス - インスタンス化不可
// 直接インスタンス化はできない Student student = new Student();
コンパイルエラー: Student is abstract; cannot be instantiated
✅ 抽象クラスの正しい使用方法
// 継承したクラスからインスタンス化 Student student = new UndergraduateStudent(); student.study(); // UndergraduateStudentの実装が実行される
コンパイル成功 - 子クラスで抽象メソッドを実装済み

抽象クラスの重要なポイント

  • インスタンス化不可: 直接 new でオブジェクト作成できない
  • 抽象メソッド: 実装を持たず、子クラスでの実装を強制
  • 具体メソッド: 通常のメソッドも定義可能（共通処理）
  • 継承の強制: 抽象メソッドがある限り、子クラスでの実装が必須
  • 設計の安全性: コンパイル時に実装漏れを検出
  • 概念の表現: 直接使用しない概念（学生、動物、図形など）の表現に最適

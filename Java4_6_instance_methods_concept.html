インスタンスメソッド vs staticメソッドの違い

🎯 インスタンスメソッド

Student tanaka = new Student("田中", 20, 3.5);
  • tanaka.showInfo() ← 田中の情報を表示
  • tanaka.updateGpa(3.8) ← 田中のGPAを更新
  • tanaka.getSummary() ← 田中の要約を取得
Student sato = new Student("佐藤", 19, 3.2);
  • sato.showInfo() ← 佐藤の情報を表示
  • sato.updateGpa(3.6) ← 佐藤のGPAを更新
  • sato.getSummary() ← 佐藤の要約を取得
public void showInfo() {
    System.out.println("名前：" + this.name);
    // thisで「このオブジェクト」を参照
}

📊 staticメソッド

Student クラス全体
  • Student.getTotalCount() ← 全学生数を取得
  • Student.createDefaultStudent() ← デフォルト学生作成
  • Student.compareGpa(s1, s2) ← GPA比較
↑
どのオブジェクトとも関係ない
クラス全体の処理
public static int getTotalCount() {
    return totalStudents;
    // thisは使えない（特定のオブジェクトがないため）
}

💡 使用例での違い

インスタンスメソッド：
Student student = new Student(...);
student.showInfo(); ← オブジェクト経由で呼び出し
staticメソッド：
Student.getTotalCount(); ← クラス名で直接呼び出し
オブジェクトを作らなくても使える

🔑 判断基準

インスタンスメソッドにする場合
✅ 特定のオブジェクトのデータを使う
✅ thisキーワードが必要
✅ オブジェクトの状態を変更する
✅ 「この学生の～」という処理
staticメソッドにする場合
✅ クラス全体に関わる処理
✅ 汎用的なユーティリティ機能
✅ オブジェクトなしでも使いたい
✅ 「学生全体の～」という処理
💫 重要：インスタンスメソッドは「誰が何をするか」を明確にする！
tanaka.study() → 田中が勉強する / sato.takeExam() → 佐藤が試験を受ける

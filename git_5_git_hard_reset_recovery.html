--hard Reset による完全復旧の実行過程
実験用コミットの作成
echo "# 実験用のテストファイル" > test_file.txt
git add test_file.txt
git commit -m "実験用コミット: テストファイル追加"
[main c3d4e5f] 実験用コミット: テストファイル追加 1 file changed, 1 insertion(+) create mode 100644 test_file.txt
追加の実験用変更
echo "追加内容" >> test_file.txt
git add test_file.txt
git commit -m "実験用コミット2: テストファイル更新"
[main d4e5f6a] 実験用コミット2: テストファイル更新 1 file changed, 1 insertion(+)
--hard reset の実行
git reset --hard a7b8c9d
HEAD is now at a7b8c9d 機能拡張完了: ファイル操作の統合管理システム

⚠️
--hard reset の重要な注意点
• 指定したコミット以降の変更が完全に削除される
• ファイルの内容も元に戻り、未保存の変更は失われる
• 実験用コミットは履歴からも削除され、簡単には復旧できない
• 使用前に必ずバックアップまたはreflogで復旧可能性を確認する
🔄 削除したコミットの復旧方法

1. reflogで履歴確認

git reflog

削除されたコミットIDを特定

2. コミットIDで復旧

git reset --hard d4e5f6a

削除したコミットを復旧

💡 ベストプラクティス: --soft → --mixed → --hard の順序で段階的に試す

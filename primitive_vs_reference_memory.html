プリミティブ型 vs 参照型のメモリ構造
プリミティブ型（基本型）
int a = 10;
int b = 10;
変数 a
10
変数 b
10
📦 変数に直接値を保存
参照型（String等）
String s1 = new String("Hello");
String s2 = new String("Hello");
変数 s1
住所: 0x1001
実際のオブジェクト (0x1001)
"Hello"
変数 s2
住所: 0x2001
実際のオブジェクト (0x2001)
"Hello"
📍 変数にオブジェクトの場所（住所）を保存

== 演算子の動作の違い

プリミティブ型： 値同士を比較
→ a == b は 10 == 10 なので true

参照型： 住所同士を比較
→ s1 == s2 は 0x1001 == 0x2001 なので false

文字列の内容を比較したい場合：
→ s1.equals(s2) を使用（中身の "Hello" 同士を比較）

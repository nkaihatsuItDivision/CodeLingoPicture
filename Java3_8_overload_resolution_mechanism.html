🔍 オーバーロード解決の仕組み

Javaコンパイラが適切なメソッドを自動選択する過程を見てみよう

1
メソッド呼び出し
search("Hello World", "world", true)
引数の分析:
• 第1引数: String ("Hello World")
• 第2引数: String ("world")
• 第3引数: boolean (true)
2
候補メソッド検索

コンパイラが「search」という名前のメソッドを全て検索します

検索条件:
• メソッド名: "search"
• 引数の数: 3個
• 引数の型: String, String, boolean
📋 利用可能なメソッド一覧
search(String, String)
基本的な文字列検索
❌ 引数の数が一致しない (2個 ≠ 3個)
search(String, String, boolean)
大文字小文字を制御する検索
✅ 完全一致！
search(String, String, int)
開始位置を指定する検索
❌ 第3引数の型が一致しない (int ≠ boolean)
search(String, String...)
複数キーワード検索
❌ 第3引数がboolean型に適用できない
⬇️
🎯 解決完了！
コンパイラが最適なメソッドを自動選択しました
search(String text, String keyword, boolean ignoreCase)

このメソッドが実行され、大文字小文字を無視した検索が行われます

💡 重要なポイント

  • 完全自動：プログラマーが意識する必要なし
  • 型安全：コンパイル時にエラーをチェック
  • 最適選択：最も適合するメソッドを選択
  • 直感的：引数を見れば動作が予想できる

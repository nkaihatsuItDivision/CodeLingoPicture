オブジェクト指向の3つのメリット
🗂️
1. 整理整頓
関連するデータと処理が一箇所にまとまり、
コードの構造が分かりやすくなります
データと処理が統合
役割が明確
管理しやすい
📊 整理整頓の効果
❌ 手続き型
• データがバラバラ
• 処理も別々
• 関連性が不明
✅ OOP
• データと処理が統合
• 責任が明確
• 一目で理解可能
// 整理された構造
class Student {
// データ
String name;
int score;
// 関連する処理
String getGrade() { ... }
void introduce() { ... }
}
♻️
2. 再利用性
一度作ったクラスは何度でも使え、
同じコードを繰り返し書く必要がありません
同じクラスで複数作成
他プロジェクトでも利用
開発効率UP
♻️ 再利用の威力
// 1つのクラスから複数のオブジェクト
Student student1 = new Student();
Student student2 = new Student();
Student student3 = new Student();
// 別のプロジェクトでも同じクラスを使用
// → コードの再利用で開発時間短縮
❌ 従来
• 毎回同じコードを書く
• 修正も毎回必要
• 開発時間がかかる
✅ OOP
• 一度作れば何度でも使用
• 修正は一箇所だけ
• 開発時間大幅短縮
💡
3. 理解しやすさ
現実世界の「モノ」として考えられるため、
プログラムの動作が直感的に理解できます
現実世界と対応
直感的な理解
チーム開発に最適
💡 理解しやすさの実感
// 現実世界の「学生」と対応
Student tanaka = new Student();
tanaka.name = "田中";
tanaka.score = 85;
// 「田中さんが自己紹介する」
tanaka.introduce();
// → 自然で理解しやすい！
❌ 抽象的
• process1(), process2()
• 何をしているか不明
• 理解に時間がかかる
✅ 直感的
• student.introduce()
• 「学生が自己紹介」
• 一目で理解可能
🎯 オブジェクト指向の本質
これら3つのメリットにより、「保守しやすく」「理解しやすく」「再利用しやすい」プログラムを作ることができます。

特に大規模な開発やチーム開発では、オブジェクト指向の恩恵を強く実感できるでしょう。
あなたも Week 3 の StringUtils クラスで、これらのメリットを既に体験しているはずです！

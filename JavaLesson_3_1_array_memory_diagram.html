🔗 連続したメモリ領域に配置（4バイト × 5要素 = 20バイト）

🚀 なぜ配列は高速アクセスできるの？

numbers[2] にアクセスする場合（ハイライトされた要素）：

配列の開始アドレス + (インデックス × データサイズ)
= 1000番地 + (2 × 4バイト) = 1008番地

✨ 計算一回で直接アクセス！

• インデックスがあれば、どの要素でも同じ速度でアクセス可能

• メモリ上で連続して配置されているため効率的

• この仕組みが配列の最大の利点です

📝 重要ポイント

• 連続メモリ配置: 要素が隣り合ったメモリ位置に配置

• 固定サイズ: 作成時にサイズが決まり、後から変更不可

• インデックスアクセス: 番号で直接要素にアクセス

• 高速アクセス: O(1)の時間計算量で要素取得

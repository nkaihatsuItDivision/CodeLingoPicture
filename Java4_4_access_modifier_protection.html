アクセス修飾子による保護
❌ 危険：publicフィールド
BadStudent クラス
public String name;
public int score;
外部からの自由なアクセス
student.name = "";
student.score = -50;
student.score = 999;
⚠️
問題点：
  • 不正なデータ設定が可能
  • データの整合性が保てない
  • バグの原因となりやすい
  • セキュリティリスクが高い
✅ 安全：privateフィールド
GoodStudent クラス
private String name;
private int score;
+ setScore(int score) メソッド
+ getName() メソッド
外部からの直接アクセスは不可
student.score = -50; // エラー！
student.setScore(85); // 安全！
🛡️
利点：
  • データの整合性を保護
  • 不正な値の設定を防止
  • メソッドで検証可能
  • セキュリティが向上

🔒 privateフィールドの保護メカニズム

privateフィールドは、クラスの外部から直接アクセスできません。データの読み取りや変更には、専用のメソッドを通す必要があります。

これにより、データの検証や整合性チェックを行うことができ、不正なデータの設定を防ぐことができます。

💡 重要ポイント：フィールドは原則private！メソッドで安全にアクセス！

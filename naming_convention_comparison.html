変数の命名規則 - 良い例と悪い例
✅ 良い例（推奨）
int age = 25;
◎ 理由：
• 短くて分かりやすい
• 年齢を表すことが明確
• 英語で一般的な単語
String firstName = "太郎";
◎ 理由：
• キャメルケース（firstName）
• 意味が具体的
• 読みやすい
int testScore = 85;
◎ 理由：
• 何の点数か明確
• 適切な長さ
• 誤解の余地がない
String emailAddress = "test@example.com";
◎ 理由：
• 省略せず完全な単語
• キャメルケース使用
• 用途が明確
❌ 悪い例（非推奨）
int a = 25;
✗ 問題点：
• 何の変数か不明
• 短すぎて意味不明
• 後で読み返せない
String 名前 = "太郎";
✗ 問題点：
• 日本語使用（非推奨）
• 文字化けの可能性
• チーム開発で問題
int user-age = 25;
✗ 問題点：
• ハイフン使用不可
• コンパイルエラー
• 文法違反
String TESTSCOREOFSTUDENTINMATH = "85";
✗ 問題点：
• 長すぎて読みにくい
• 全部大文字（定数風）
• タイプミスしやすい
🐫 キャメルケース（camelCase）とは？
最初の文字は小文字、次の単語の最初は大文字にする記法
firstName
first + Name
testScore
test + Score
userEmailAddress
user + Email + Address
totalStudentCount
total + Student + Count
🔄 改善の例
int a = 20;
❌ 意味不明
→
int age = 20;
✅ 年齢とわかる
String str = "田中";
❌ 抽象的
→
String userName = "田中";
✅ ユーザー名とわかる
int num = 85;
❌ 何の数字？
→
int mathScore = 85;
✅ 数学の点数とわかる

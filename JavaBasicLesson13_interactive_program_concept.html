🔄 固定値プログラム vs 対話型プログラム

❌ 固定値プログラム（従来）
public class FixedCalculator { public static void main(String[] args) { // 値がプログラム内で固定 int a = 10; int b = 5; int result = a + b; System.out.printf("%d + %d = %d%n", a, b, result); } }
10 + 5 = 15

🚫 問題点

  • 値が固定されている
  • コード変更が必要
  • 再コンパイルが必要
  • ユーザーが自由に使えない
  • 実用性が低い
→
✅ 対話型プログラム（Scanner使用）
import java.util.Scanner; public class InteractiveCalculator { public static void main(String[] args) { Scanner scanner = new Scanner(System.in); System.out.print("1つ目の数値: "); int a = scanner.nextInt(); System.out.print("2つ目の数値: "); int b = scanner.nextInt(); int result = a + b; System.out.printf("%d + %d = %d%n", a, b, result); scanner.close(); } }
1つ目の数値: 25 2つ目の数値: 13 25 + 13 = 38

🌟 利点

  • ユーザーが自由に値を入力
  • 同じプログラムで様々な計算
  • リアルタイムで結果確認
  • 実用的なアプリケーション
  • ユーザー体験の向上

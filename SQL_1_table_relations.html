テーブル構造とリレーションの基本概念
Week 5学生管理システムを例に理解するデータベースの基本
❌ Week 5での配列管理
// Week 5システムでのデータ管理 Map<String, Student> students = new HashMap<>(); List<Grade> gradeHistory = new ArrayList<>(); // 学生データの設定 students.put("S001", new Student("田中太郎", "CS")); students.put("S002", new Student("山田花子", "ML")); // 成績データの設定 gradeHistory.add(new Grade("S001", "数学", 85)); gradeHistory.add(new Grade("S001", "プログラミング", 92));

⚠️ 配列管理の問題点

  • プログラム終了でデータ消失
  • 学生と成績の関連が手動管理
  • データ整合性チェックが困難
  • 複雑な検索処理が必要
  • メモリ使用量の制限
✅ データベーステーブル管理
-- 学生テーブルの作成 CREATE TABLE students ( student_id VARCHAR(10) PRIMARY KEY, name VARCHAR(50), major VARCHAR(50) ); -- 成績テーブルの作成（外部キーで関連付け） CREATE TABLE grades ( grade_id INT AUTO_INCREMENT PRIMARY KEY, student_id VARCHAR(10), subject VARCHAR(100), score INT, FOREIGN KEY (student_id) REFERENCES students(student_id) );

✅ データベースの解決策

  • 永続的なデータ保存
  • 外部キーによる自動関連管理
  • 制約による整合性保証
  • 高速なSQL検索
  • 大容量データ対応
📊 テーブル構造の基本要素
🗄️
データベース
関連するテーブルをまとめて管理する入れ物
📋
テーブル
データを行と列の表形式で格納する構造
📝
レコード（行）
1つの完全なデータセット（例：1人の学生情報）
📊
カラム（列）
データの属性を定義（例：名前、年齢、専攻）
👥 students テーブル
student_id name major
S001 田中太郎 CS
S002 山田花子 ML
S003 佐藤一郎 AI
📊 grades テーブル
grade_id student_id subject score
1 S001 数学 85
2 S001 プログラミング 92
3 S002 機械学習 88
🔗 テーブル間の関係性
↕️
students.student_id と grades.student_id が連携
一人の学生は複数の成績を持つ（1対多の関係）
外部キー制約により、存在しない学生の成績は登録不可
🔗 リレーション（関係）の仕組み
👥 Students
PK student_id
name
major
←→
1 : N
📊 Grades
PK grade_id
FK student_id
subject
score
1人の学生（Students） : 複数の成績（Grades）
✨ データベーステーブルの利点
💾
永続性: プログラム終了後もデータが保持され、次回起動時に自動的に利用可能
🔗
関係性: 外部キーにより学生と成績の関連が自動的に管理・保証される
🔍
高速検索: インデックスにより大量データでも瞬時に目的の情報を取得
✅
整合性: 制約により不正なデータの登録を防ぎ、データの品質を自動保証
👥
同時アクセス: 複数のユーザーが安全に同時にデータを操作可能
📈
拡張性: メモリ制限を超えた大容量データの効率的な管理が可能

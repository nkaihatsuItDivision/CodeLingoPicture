UPDATE文の安全性：WHERE句の重要性

❌ 危険な操作：WHERE句なし
-- 全学生の年齢が25歳に変更される！
UPDATE student
SET age = 25;
影響範囲：全レコード (8件すべて)
結果：すべての学生が25歳に！
復旧：不可能（元データ失われる）
✅ 安全な操作：WHERE句あり
-- 田中太郎のみ年齢変更
UPDATE student
SET age = 25
WHERE student_id = 'S001';
影響範囲：1レコードのみ
結果：田中太郎のみ25歳に変更
復旧：特定レコードなので対応可能
💀 WHERE句なし操作の深刻な影響
  • データ損失リスク：元の年齢データが完全に失われ、復旧不可能
  • 業務への影響：学生管理システム全体が使用不能になる
  • 信頼性の失墜：データの整合性が崩れ、システムへの信頼が失われる
  • 経済的損失：データ復旧作業、業務停止による損失が発生
✅ UPDATE文の安全な実行手順
  • 1事前確認：SELECT文で更新対象レコードを確認
  • 2WHERE句必須：必ず条件を指定して対象を限定
  • 3トランザクション：BEGIN～COMMITで安全性を確保
  • 4事後確認：更新後にSELECTで結果を検証

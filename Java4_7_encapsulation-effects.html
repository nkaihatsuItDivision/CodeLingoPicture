📈 カプセル化の実践効果

📚 Bookクラスの設計 

public class Book {
    // カプセル化：全てprivate
    private String title;
    private String author;
    private double price;
    private boolean available;
    
    // 検証付きメソッド
    public void updatePrice(double newPrice) {
        if (newPrice >= 0) {
            this.price = newPrice;
        } else {
            System.out.println("無効な価格");
        }
    }
}

🔒 安全な使用方法 

Book book = new Book("Java入門", 
                   "山田", 2800, 350);

// ✅ 安全な操作
book.updatePrice(2500);  // 正常更新
book.borrowBook();       // 貸出処理
String info = book.getBookInfo();

// ❌ 不正操作は不可
// book.price = -100; // エラー

// ✅ 不正値は自動拒否
book.updatePrice(-100);  // "無効な価格"
🛡️

データ保護

privateフィールドにより外部からの不正アクセスを完全ブロック

✅

入力検証

メソッド内での検証により不正値の設定を自動防止

🔧

保守性向上

内部実装変更時の外部コードへの影響を最小限に抑制

🎯

使いやすさ

明確なインターフェースにより直感的な操作を実現

⚙️ Controller実装のポイント
🎯
@PathVariable
URL中の変数を取得するアノテーション
動的なパスパラメータの処理に使用
@GetMapping("/{id}")
public ResponseEntity<ApiResponse> getStudent(
    @PathVariable Long id) {
    // id変数にURL中の{id}値が設定される
    return studentService.findById(id);
}
💡 実用価値：個別リソースへのアクセスが簡単
📦
@RequestBody
HTTPリクエストのボディを取得
JSON形式のデータを自動でオブジェクトに変換
@PostMapping
public ResponseEntity<ApiResponse> createStudent(
    @RequestBody Student student) {
    // JSONが自動でStudentオブジェクトに変換
    return studentService.create(student);
}
💡 実用価値：複雑なデータも簡単に受け取り可能
🔍
@RequestParam
クエリパラメータを取得
検索条件やページング情報の受け取りに使用
@GetMapping("/search")
public ResponseEntity<ApiResponse> searchStudents(
    @RequestParam String name,
    @RequestParam(defaultValue = "0") int page) {
    // ?name=田中&page=1 → name="田中", page=1
    return studentService.search(name, page);
}
💡 実用価値：検索機能の実装が容易
✅
@Valid
入力データの妥当性検証
Bean Validationと組み合わせて自動検証
@PostMapping
public ResponseEntity<ApiResponse> createStudent(
    @Valid @RequestBody Student student,
    BindingResult result) {
    if (result.hasErrors()) {
        return ResponseEntity.badRequest();
    }
    return studentService.create(student);
}
💡 実用価値：入力チェックの自動化
📊 ResponseEntity活用パターン

🎯 成功レスポンス

return ResponseEntity.ok(
    ApiResponse.success(data)
);

ステータス：200 OK

➕ 作成成功

return ResponseEntity.status(201)
    .body(ApiResponse.success(data)
);

ステータス：201 Created

⚠️ 入力エラー

return ResponseEntity.badRequest()
    .body(ApiResponse.error(msg)
);

ステータス：400 Bad Request

🔍 データ未発見

return ResponseEntity.notFound()
    .build();

ステータス：404 Not Found

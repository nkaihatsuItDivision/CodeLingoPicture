🔄 コンストラクタオーバーロードの使い分け
📋 オーバーロードとは
同じクラスに複数のコンストラクタを定義すること
引数の種類や数が異なれば複数定義できます
🎯 コンストラクタ1：デフォルト
public Student() { this.name = ""; this.age = 18; this.gpa = 0.0; }
📝 使用例
Student student = new Student();
全てデフォルト値で初期化
👤 コンストラクタ2：名前のみ
public Student(String name) { this(); this.name = name; }
📝 使用例
Student student = new Student("田中太郎");
名前を指定、他はデフォルト値
👤📅 コンストラクタ3：名前と年齢
public Student(String name, int age) { this(name); this.age = age; }
📝 使用例
Student student = new Student("佐藤花子", 19);
名前と年齢を指定、GPAはデフォルト
👤📅📊 コンストラクタ4：全項目
public Student(String name, int age, double gpa) { this(name, age); this.gpa = gpa; }
📝 使用例
Student student = new Student("山田次郎", 21, 3.5);
全ての項目を明示的に指定
情報なし
→
名前のみ
→
名前+年齢
→
全項目指定
🎯 this() による効率的な実装
🔄
コンストラクタ呼び出し
this() で他のコンストラクタを呼び出し
♻️
コード再利用
重複した初期化処理を避ける
🛡️
安全性確保
共通処理の実行を保証
📈
保守性向上
変更時の影響範囲を最小化
✨ オーバーロードの利点
✅
柔軟性：様々な初期化パターンに対応
✅
使いやすさ：必要な情報だけ指定可能
✅
段階的初期化：情報が揃った時点で使用開始
✅
デフォルト値活用：未指定項目は安全な初期値
✅
コード効率化：this()による重複回避
✅
保守性：変更時の影響範囲を限定
🎓 重要なルール
1. this() は最初の行に書く必要があります
2. 一つのコンストラクタで使えるthis()は一つだけ
3. 循環呼び出し禁止（A→B→A のような呼び出し）
4. 引数の種類・数が異なれば複数定義可能

📁 新規ファイル追加の流れ

AuditStudentクラスの新規作成を通じて新機能開発の管理を実践

1
新しいクラスファイルの作成
システム拡張のため、聴講生を管理するAuditStudentクラスを新規作成
📄 AuditStudent.java
/** * 聴講生クラス * Week 6レッスン4で新規追加 */ public class AuditStudent extends Person { private String auditCourse; private int attendanceCount; // ... コンストラクタ、メソッド }
📄 AuditStudentTest.java
/** * AuditStudentクラスのテストプログラム */ public class AuditStudentTest { public static void main(String[] args) { // テストコード } }
⬇️
2
新規ファイルの検出
Gitが新規ファイルをどのように認識するかを確認
$ git status
On branch main
Untracked files:
  (use "git add <file>..." to include in what will be committed)

        AuditStudent.java
        AuditStudentTest.java

nothing added to commit but untracked files present
📝 重要なポイント
Untracked files として新規ファイルが認識されます
まだGitの管理下にない状態を「追跡されていない」と表現します
⬇️
3
複数新規ファイルの管理
複数の新規ファイルを効率的にステージングに追加
$ git add AuditStudent.java AuditStudentTest.java
または
$ git add *.java

追加前

Untracked files:
        AuditStudent.java
        AuditStudentTest.java

追加後

Changes to be committed:
        new file:   AuditStudent.java
        new file:   AuditStudentTest.java
⬇️
4
新規ファイルをコミット
新機能の追加として適切なメッセージでコミット
$ git commit -m "新機能追加: 聴講生（AuditStudent）クラスとテストプログラムを追加 - AuditStudent.java: 聴講生を管理するクラス - Person継承で基本情報管理 - 聴講講座、出席回数、継続状況の管理 - AuditStudentTest.java: 聴講生機能のテストプログラム"
[main h8i9j0k] 新機能追加: 聴講生（AuditStudent）クラスとテストプログラムを追加
 2 files changed, 145 insertions(+)
 create mode 100644 AuditStudent.java
 create mode 100644 AuditStudentTest.java
✨ 注目ポイント
create mode 100644 で新規ファイル作成が記録されます
修正ファイルとは異なる履歴として管理されます

🎊 新規ファイル追加の管理が完了しました！

システム拡張時の新機能ファイルを安全にGitで管理する技術をマスターしました。
新規作成 → 検出確認 → ステージング → コミットの流れで、チーム開発での新機能追加が可能です！

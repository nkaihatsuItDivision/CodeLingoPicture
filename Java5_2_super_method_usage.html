super.メソッド名()の2つの活用パターン
❌ 置換型（推奨しない）
@Override
public void displayInfo() {
    // 親クラスの機能を使わずに完全に再実装
    System.out.println("学生ID: " + studentId);
    System.out.println("氏名: " + name);
    System.out.println("年齢: " + age + "歳");
    System.out.println("専攻: " + major);
    System.out.println("必要単位数: " + requiredCredits);
}
1 親クラスのメソッドを呼び出さない
2 すべてのコードを子クラスで再実装
3 重複コードが発生
✅ 拡張型（推奨）
@Override
public void displayInfo() {
    // 1. 親クラスの基本機能を活用
    super.displayInfo();
    
    // 2. 子クラス固有の情報を追加
    System.out.println("必要単位数: " + requiredCredits);
    System.out.println("学生種別: 学部生");
}
1 super.displayInfo()で親の機能を実行
2 子クラス固有の処理を追加
3 親＋子の統合された機能を実現
VS
❌ 問題点
  • コードの重複が発生
  • 親クラス変更時の影響が大きい
  • 保守性が低下
  • 継承の利点を活かせない
✅ 利点
  • コードの重複を完全に回避
  • 親クラスの変更が自動反映
  • 保守性が大幅に向上
  • 継承の利点を最大活用

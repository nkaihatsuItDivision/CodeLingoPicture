エラーハンドリングの重要性
✗
エラーハンドリングなし
int age = scanner.nextInt(); System.out.println("年齢: " + age);
ユーザー入力: "abc"
↓
Exception in thread "main"
java.util.InputMismatchException
プログラムクラッシュ！
✓
エラーハンドリングあり
try { int age = scanner.nextInt(); System.out.println("年齢: " + age); } catch (InputMismatchException e) { System.out.println("数値を入力してください"); }
ユーザー入力: "abc"
↓
数値を入力してください
プログラム継続実行

🎯 エラーハンドリングの効果

  • プログラムの安定性: 予期しない入力でもクラッシュしない
  • ユーザビリティ向上: 分かりやすいエラーメッセージ
  • プロフェッショナルな品質: 実用的なアプリケーションレベル
  • 継続利用可能: エラー後もプログラムを使い続けられる
  • デバッグ支援: 問題の原因が特定しやすい

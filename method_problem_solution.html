メソッドが解決する重要な問題

😰 for文では解決できない問題
// 学生情報を表示するプログラム
public class StudentInfo {
    public static void main(String[] args) {
        // 太郎さんの情報を表示
        System.out.println("=== 学生情報 ===");
        System.out.println("名前: 太郎");
        System.out.println("年齢: 20");
        System.out.println("成績: 85点");
        System.out.println();

        // 花子さんの情報を表示
        System.out.println("=== 学生情報 ===");
        System.out.println("名前: 花子");
        System.out.println("年齢: 19");
        System.out.println("成績: 92点");
        System.out.println();

        // 次郎さんの情報を表示
        System.out.println("=== 学生情報 ===");
        System.out.println("名前: 次郎");
        System.out.println("年齢: 21");
        System.out.println("成績: 78点");
        System.out.println();
    }
}

❌ 何が問題なのか？

• 同じ処理が3回も書かれている → コードが長い
• データが違うのでfor文は使えない → 繰り返し処理では解決不可
• 修正が大変 → 表示形式を変更する場合、3箇所を直す必要
• バグが生まれやすい → 修正し忘れや間違いが発生しやすい
↓
😊 メソッドによる解決
// メソッドを使った改善版
public class StudentInfoImproved {
    public static void main(String[] args) {
        // スッキリ！メソッドを呼び出すだけ
        showStudentInfo("太郎", 20, 85);
        showStudentInfo("花子", 19, 92);
        showStudentInfo("次郎", 21, 78);
    }
}
// 学生情報を表示するメソッド
public static void showStudentInfo(String name, int age, int score) {
    System.out.println("=== 学生情報 ===");
    System.out.println("名前: " + name);
    System.out.println("年齢: " + age);
    System.out.println("成績: " + score + "点");
    System.out.println();
}

✅ メソッドのメリット

1
コードの重複解消： 同じ処理を1箇所にまとめられる
2
保守性の向上： 修正は1箇所だけ。バグも減る
3
再利用性： 一度作ったメソッドを何度でも使える
4
可読性向上： 処理の意味が明確になり、理解しやすい

📊 Scanner入力メソッド完全比較表

適切なメソッドを選択して安全で使いやすい入力処理を実現しよう

メソッド 戻り値型 用途・特徴 使用例 注意点
nextLine() String
  • 改行まで全て取得
  • スペース含む文字列OK
  • 日本語入力OK
  • 最も安全で推奨
String name = scanner.nextLine();
// 入力: "山田 太郎"
// 結果: "山田 太郎"
  • 空文字列も有効な入力
  • 数値変換が必要な場合あり
next() String
  • 最初の単語のみ取得
  • スペースで区切り
  • 限定的な用途
String word = scanner.next();
// 入力: "Hello World"
// 結果: "Hello"
// ("World"は残る)
  • スペース以降は取得されない
  • 残りデータが入力バッファに残る
  • 一般的には非推奨
nextInt() int
  • 整数値を直接取得
  • 型変換不要
  • 数値計算に便利
int age = scanner.nextInt();
// 入力: "25"
// 結果: 25 (int型)
  • 数値以外でエラー発生
  • 改行文字が残る
  • hasNextInt()での事前チェック推奨
nextDouble() double
  • 小数値を直接取得
  • 型変換不要
  • 精密な計算に最適
double price = scanner.nextDouble();
// 入力: "1980.5"
// 結果: 1980.5 (double型)
  • 数値以外でエラー発生
  • 改行文字が残る
  • hasNextDouble()での事前チェック推奨

重要な注意点：nextInt() → nextLine() の組み合わせ

❌ 問題のあるコード

int age = scanner.nextInt();
String name = scanner.nextLine();
// nameが空文字になる問題

✅ 正しいコード

int age = scanner.nextInt();
scanner.nextLine(); // 改行文字を消去
String name = scanner.nextLine();

理由: nextInt()は数値のみを読み取り、改行文字を残すため、続くnextLine()が空文字を返してしまいます。

使い分けの推奨パターン

🥇 最優先：nextLine() + 文字列変換
全ての入力をnextLine()で受け取り、必要に応じてInteger.parseInt()やDouble.parseDouble()で変換する方法。最も安全で予測可能。
🥈 次善：型別メソッド + hasNext系チェック
nextInt()やnextDouble()を使用するが、必ずhasNextInt()やhasNextDouble()で事前チェックを行う方法。
🏅 学習用：段階的に理解
まず基本的な使い方を理解し、エラーを体験してから安全な処理を学ぶアプローチ。今回のレッスンはこの方針。

💡 プロのアドバイス: 実用的なプログラムでは一貫してnextLine()を使用し、
必要に応じて型変換することを強く推奨します。

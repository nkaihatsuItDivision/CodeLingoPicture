try-catch文の動作フローチャート
1
プログラム実行開始
2
tryブロック内のコードを実行
int age = scanner.nextInt();
3
例外が発生した？
いいえ (正常)
4a
tryブロックの続きを実行
catchブロックはスキップ
はい (例外発生)
4b
catchブロックを実行
エラー処理コード
5
try-catch文の後の処理を継続
プログラムは正常に続行
実際のコード例:

try {
    int age = scanner.nextInt(); // ← ここで例外が発生する可能性
    System.out.println("年齢: " + age);
} catch (InputMismatchException e) {
    System.out.println("数値を入力してください"); // ← 例外発生時の処理
    scanner.nextLine(); // バッファクリア
}
System.out.println("プログラム継続"); // ← 必ず実行される

🔑 重要なポイント

  • 必ず1つのパスを通る: 正常パスまたは例外パスのどちらか
  • プログラムは継続: 例外が発生してもクラッシュしない
  • 適切な処理分岐: 状況に応じた適切な処理を実行
  • バッファクリア: catch文でscanner.nextLine()を忘れずに

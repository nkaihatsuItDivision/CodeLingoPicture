主要なGitコマンド
➕
git add
git add Student.java
ワーキングディレクトリの変更をステージングエリアに追加。コミット対象として準備する。
💾
git commit
git commit -m "メッセージ"
ステージングエリアの変更をリポジトリに確定保存。履歴として永続的に記録される。
📊
git status
git status
各エリアの現在状態を確認。どのファイルが変更・ステージングされているかを表示。
🔍
git diff
git diff Student.java
ファイルの具体的な変更内容を表示。どの行が追加・削除・修正されたかを確認。
変更のライフサイクル
1
ファイル編集
Eclipse等でコードを修正・作成
2
ステージング
git addで変更をコミット準備エリアに追加
3
コミット
git commitで変更を履歴として確定
4
プッシュ
git pushでリモートリポジトリに共有

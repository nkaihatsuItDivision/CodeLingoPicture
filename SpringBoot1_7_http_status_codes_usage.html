📊 HTTPステータスコードの適切な使用
✅
2xx 成功系
200 OK
リクエストが正常に処理された
GET /api/students → 学生一覧取得成功
201 Created
新しいリソースが正常に作成された
POST /api/students → 新規学生登録成功
204 No Content
処理成功、レスポンスボディなし
DELETE /api/students/1 → 削除成功
🔄
3xx リダイレクト系
301 Moved Permanently
リソースが恒久的に移動
旧APIから新APIへの恒久移行
304 Not Modified
キャッシュされた内容が最新
If-Modified-Since ヘッダー使用時
⚠️
4xx クライアントエラー系
400 Bad Request
リクエストの形式や内容が不正
必須フィールドの不備、妥当性検証エラー
401 Unauthorized
認証が必要または認証失敗
認証トークンなし・無効
403 Forbidden
認証済みだが権限不足
管理者権限が必要な操作
404 Not Found
指定されたリソースが存在しない
存在しない学生IDでアクセス
409 Conflict
リソースの状態が競合
重複する学籍番号での登録
💥
5xx サーバーエラー系
500 Internal Server Error
サーバー内部でエラーが発生
予期しない例外、システム障害
502 Bad Gateway
上流サーバーからの不正応答
外部API呼び出し失敗
503 Service Unavailable
サーバーが一時的に利用不可
メンテナンス中、過負荷状態

💡 クライアントが一貫して処理可能
Global Exception Handler
@ControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(ValidationException.class)
    public ResponseEntity handleValidation() {
        return ResponseEntity.badRequest();
    }
}

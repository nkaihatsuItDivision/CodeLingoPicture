✅ メソッドチェーン（簡潔）
Student student = new Student("田中", 18, 3.0);

student.addSubject("数学")
       .setGpa(3.8)
       .setAge(20)
       .showInfo();

// 1つの文で完結、流れが明確

🌟 メソッドチェーンの利点

📝 簡潔性
複数の処理を1つの文で表現できる。変数名の繰り返しが不要で、コードが短くなる。
👁️ 可読性
処理の流れが一目で分かる。上から下に読むだけで処理の順序が理解できる。
🔧 メンテナンス性
処理の追加・削除が簡単。チェーンの途中に新しいメソッドを挿入するのも容易。
🎯 直感性
「学生を作って、科目を追加して、GPAを設定して、表示する」という流れが自然。

🛠️ メソッドチェーン実装ガイド

ステップ1: 戻り値をStudentにする
// voidの代わりにStudentを戻り値にする
public Student addSubject(String subject) {
    // 処理を実行
    this.subjects.add(subject);
    return this; // 重要！
}
ステップ2: 自分自身（this）を返す
public Student setGpa(double gpa) {
    this.gpa = gpa;
    return this; // 同じオブジェクトを返す
}
ステップ3: チェーンの終端はvoidでもOK
public void showInfo() {
    // 表示処理（戻り値不要）
    System.out.println(...);
    // return文なし
}
⚠️ 注意点
・全てのメソッドをチェーン対応にする必要はない
　表示系メソッド（showInfo()）は通常voidのまま
・適度な改行で可読性を保つ
　長すぎるチェーンは逆に読みにくい
・意味のある処理のまとまりでチェーンする
　関連性のない処理は分ける
🔗 メソッドチェーンの本質
「return this;」で同じオブジェクトを次のメソッドに渡し続ける仕組み
オブジェクトに対する一連の操作を自然な流れで表現する現代的な設計パターン

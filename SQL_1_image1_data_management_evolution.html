配列・ArrayList方式
Map<String, Student> students = new HashMap<>(); List<Grade> gradeHistory = new ArrayList<>(); // プログラム終了でデータ消失 students.put("S001", new Student("田中太郎"));

❌ 問題点

  • プログラム終了でデータ消失
  • 複数PC間でデータ共有不可
  • 大量データで検索が遅い
  • データ整合性の手動管理
  • バックアップ・復旧が困難
→
ファイル管理方式
// students.txt に保存 PrintWriter writer = new PrintWriter("students.txt"); writer.println("S001,田中太郎,20,CS"); // 読み込み時に全解析が必要 BufferedReader reader = new BufferedReader(...);

❌ 新たな問題

  • 同時アクセスでデータ競合
  • 複雑な継承構造の保存困難
  • 検索で全ファイル読込必要
  • データ整合性制御が複雑
  • ファイル破損リスク
→
データベース方式
-- 高速で安全なデータ管理 INSERT INTO students VALUES ('S001', '田中太郎', 20, 'CS'); -- 複雑な検索も瞬時に実行 SELECT s.name, AVG(g.score) FROM students s JOIN grades g ON s.id = g.student_id WHERE s.major = 'CS' GROUP BY s.name;

✅ 解決される問題

  • データの永続的な保存保証
  • 複数ユーザーの安全な同時アクセス
  • 高速な検索・集計処理
  • 自動的なデータ整合性確保
  • 完全なバックアップ・復旧機能

🔍 複合変更管理

修正・追加・削除・リネームを組み合わせた実践的な変更管理

📋 シナリオ設定：システム拡張の複合開発

1. 既存ファイル修正

Person.javaにメソッド追加

2. 新規ファイル作成

StudentUtils.javaユーティリティクラス

3. ファイル名変更

AuditStudentTest.java → AuditStudentDemo.java

🔄 変更実行前

$ git status
On branch main
nothing to commit, working tree clean
✨ クリーンな状態

⚡ 複合変更実行後

$ git status
On branch main

Changes to be committed:
        renamed:    AuditStudentTest.java → AuditStudentDemo.java

Changes not staged for commit:
        modified:   Person.java

Untracked files:
        StudentUtils.java
📊 git status の詳細解読
1

Changes to be committed

ステージング済みの変更
renamed: ファイル名変更が検出
2

Changes not staged

ステージングされていない変更
modified: 既存ファイルの修正
3

Untracked files

管理されていない新規ファイル
新規作成されたファイル
⚙️ 段階的ステージング処理
# Person.javaの変更を確認
$ git diff Person.java

# Person.javaをステージング
$ git add Person.java

# StudentUtils.javaもステージング
$ git add StudentUtils.java

# 最終状態確認
$ git status

📋 全変更ステージング後の状態

$ git status
On branch main
Changes to be committed:
        modified:   Person.java
        renamed:    AuditStudentTest.java → AuditStudentDemo.java
        new file:   StudentUtils.java
✨ 重要なポイント
• modified: 既存ファイルの修正
• renamed: ファイル名の変更
• new file: 新規ファイルの追加

すべて異なる種類の変更が1つのコミットにまとめられます

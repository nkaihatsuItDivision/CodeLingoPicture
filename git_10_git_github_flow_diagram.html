🔄 Git/GitHub 操作フロー
Pull Request を使ったチーム開発の完全ワークフロー
レッスン6.10: Fork → 開発 → PR → レビュー → マージの一連の流れ
📋 Phase 1: 初期設定・Fork・クローン
1
🍴
Fork実行
GitHubで元リポジトリをFork
➜
2
📥
Clone実行
Forkしたリポジトリをローカルに複製
git clone [Fork URL]
➜
3
🔗
upstream設定
元リポジトリをupstreamに追加
git remote add upstream [元URL]
💻 Phase 2: ブランチ作成・機能開発・プッシュ
4
🌿
ブランチ作成
機能開発用ブランチを作成・切り替え
git checkout -b feature/add-operations
➜
5
✏️
機能実装
Calculator.java に四則演算機能を実装
➜
6
💾
コミット
変更をステージング・コミット
git add . && git commit -m "feat: 四則演算機能を追加"
➜
7
📤
プッシュ
ブランチをリモートにプッシュ
git push origin feature/add-operations
📝 Phase 3: Pull Request 作成・レビュー依頼
8
🔗
PR作成画面
GitHub上で "Compare & pull request"
➜
9
📋
PR詳細入力
タイトル・説明・チェックリストを詳細記入
➜
10
🚀
PR作成完了
"Create pull request" でPR #1作成
👨‍💻 開発者の作業フロー
11
⏳
レビュー待機
レビューアーからのフィードバックを待つ
13
🔧
修正対応
レビューコメントに基づいてコード修正
git commit -m "fix: レビュー指摘事項を修正"
14
📤
修正プッシュ
修正内容をプッシュして再レビュー依頼
git push origin feature/add-operations
👨‍💼 レビューアーの作業フロー
12
👀
コードレビュー
Files changedでコード変更内容を確認
12b
💬
レビューコメント
改善提案・質問・肯定的フィードバック
15
✅
承認
修正確認後、"Approve" でレビュー承認
🎯 Phase 4: マージ・統合・完了
16
⚙️
マージ戦略選択
Merge commit/Squash/Rebase選択
➜
17
🔀
マージ実行
"Merge pull request" でmainブランチに統合
➜
18
🧹
ブランチ削除
不要になった機能ブランチを削除
git branch -d feature/add-operations
➜
19
🎉
統合完了
機能がmainブランチに正式統合

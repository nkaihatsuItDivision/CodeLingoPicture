プログラムでの契約（インターフェース）
DeliveryServiceインターフェース
public interface DeliveryService { // 契約で定められた機能 void deliver(String address, String item); String getDeliveryTime(); void notifyCompletion(); void handleDamage(); }
重要なポイント
利用者は「実装方法」を知る必要がない
「契約通りの機能」が提供されることを期待

⬇️ 実装例 ⬇️
public class YamatoDelivery implements DeliveryService { @Override public void deliver(String address, String item) { System.out.println("ヤマト運輸：トラックで" + item + "を" + address + "に配送します"); } @Override public String getDeliveryTime() { return "明日の午前中"; } @Override public void notifyCompletion() { System.out.println("ヤマト運輸：配送完了をメールで通知しました"); } @Override public void handleDamage() { System.out.println("ヤマト運輸：破損補償手続きを開始します"); } }
インターフェースの重要な特徴
1. 完全な抽象化： 全てのメソッドが抽象（実装を持たない）
2. 契約の定義： 「何をするか」のみを定義、「どうやるか」は実装クラスに委ねる
3. 多重実装： 1つのクラスが複数のインターフェースを実装可能
4. 統一インターフェース： 実装方法に関係なく同じ方法で使用可能
